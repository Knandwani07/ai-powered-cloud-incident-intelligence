# Amazon Athena

This directory contains the SQL queries used to create and query the incident data warehouse. Amazon Athena enables serverless SQL analytics on incident logs stored in Amazon S3 without requiring database infrastructure.

## Purpose

- Create the incident database.
- Define the schema for incident logs stored in Amazon S3.
- Validate data ingestion.
- Analyze incident trends and frequency.

## Directory Structure

```text
athena/
│
└── athena-queries.md
```

---

# Configure Query Result Location

Configure the following S3 path as the query result location in Athena:

```text
s3://incident-intelligence-logs-[your-account-id]-ap-south-1/athena-results/
```

This location stores the output generated by Athena queries.

---

# Create Database

Creates a database named `incident_db` to organize and manage incident-related tables.

```sql
CREATE DATABASE incident_db;
```

---

# Create External Table

Creates an external table named `incidents` that maps JSON files stored in Amazon S3 to a queryable schema.

```sql
CREATE EXTERNAL TABLE incident_db.incidents (
  timestamp       string,
  task_arn        string,
  cluster         string,
  stop_code       string,
  stopped_reason  string,
  incident_type   string,
  raw_event       string
)
ROW FORMAT SERDE 'org.openx.data.jsonserde.JsonSerDe'
LOCATION 's3://incident-intelligence-logs-[your-account-id]-ap-south-1/incident-logs/'
TBLPROPERTIES ('has_encrypted_data'='false');
```

---

# Verify Data Ingestion

Retrieves the first ten records from the incident table to confirm that the logs have been successfully ingested.

```sql
SELECT *
FROM incident_db.incidents
LIMIT 10;
```

---

# Analyze Incident Trends

Groups incidents by type and counts the number of occurrences to identify the most common failures.

```sql
SELECT
    incident_type,
    COUNT(*) as count
FROM incident_db.incidents
GROUP BY incident_type
ORDER BY count DESC;
```

- Query JSON-based incident logs stored in S3.
- Analyze historical incident trends.
- Support AI-driven reporting and monitoring.
- No database servers to manage.
